# План реализации улучшений архитектуры QuestTrip-App

## Фаза 1: Базовые улучшения (Неделя 1)

### 1.1. Устранение дублирования компонентов
- [ ] Проанализировать различия между `components/` и `components/shared/`
- [ ] Объединить дублирующие компоненты в единую структуру
- [ ] Обновить импорты во всех файлах
- [ ] Удалить лишние файлы из `components/shared/`
- [ ] Создать новую структуру компонентов:
  ```
  client/src/components/
  ├── ui/           # shadcn/ui компоненты (уже есть)
  ├── layout/       # Navbar, Footer, PageWrapper
  ├── quest/        # QuestCard, QuestGrid, QuestCardSkeleton
  ├── auth/         # LoginForm, RegisterForm
  ├── common/       # EmptyState, SectionHeader
  └── index.ts      # Barrel exports
  ```

### 1.2. Безопасность: Environment variables
- [ ] Создать `.env.example` с шаблоном переменных
- [ ] Создать `.env.development` и `.env.production`
- [ ] Вынести JWT_SECRET, ADMIN_CODE в переменные окружения
- [ ] Обновить `server/index.ts` для использования process.env
- [ ] Добавить валидацию обязательных переменных при запуске

### 1.3. Система миграций базы данных
- [ ] Установить drizzle-kit как dev dependency
- [ ] Настроить `drizzle.config.ts` для миграций
- [ ] Создать начальные миграции для существующих таблиц
- [ ] Обновить `server/storage.ts` для использования миграций вместо raw SQL
- [ ] Добавить скрипты в package.json:
  - `db:generate` - генерация миграций
  - `db:migrate` - применение миграций
  - `db:studio` - просмотр БД

### 1.4. Рефакторинг routes.ts
- [ ] Создать структуру `server/routes/`
- [ ] Разделить на модули:
  - `auth.routes.ts` - аутентификация
  - `quest.routes.ts` - квесты
  - `user.routes.ts` - пользователи
  - `admin.routes.ts` - админка
  - `index.ts` - объединение
- [ ] Вынести middleware в отдельные файлы
- [ ] Создать типы для Request/Response

## Фаза 2: Архитектурные улучшения (Неделя 2)

### 2.1. Сервисный слой
- [ ] Создать `server/services/` директорию
- [ ] Реализовать сервисы:
  - `auth.service.ts` - логика аутентификации
  - `quest.service.ts` - бизнес-логика квестов
  - `user.service.ts` - управление пользователями
  - `payment.service.ts` - платежи (заглушка)
- [ ] Вынести бизнес-логику из `routes.ts` в сервисы
- [ ] Создать интерфейсы для сервисов

### 2.2. Усиление валидации
- [ ] Расширить Zod схемы в `shared/schema.ts`
- [ ] Добавить кастомные валидаторы:
  - Валидация email
  - Валидация пароля (сложность)
  - Валидация цены квеста
  - Валидация URL изображений
- [ ] Создать middleware `validateRequest` для автоматической валидации
- [ ] Добавить DTO классы для сложных операций

### 2.3. Обработка ошибок
- [ ] Создать `server/errors/` директорию
- [ ] Определить классы ошибок:
  - `ValidationError` - ошибки валидации
  - `AuthenticationError` - ошибки аутентификации
  - `AuthorizationError` - ошибки авторизации
  - `NotFoundError` - ресурс не найден
  - `BusinessError` - бизнес-логика
- [ ] Создать глобальный обработчик ошибок
- [ ] Стандартизировать формат ответов об ошибках

### 2.4. Базовое кэширование
- [ ] Добавить in-memory кэш для часто запрашиваемых данных
- [ ] Реализовать кэширование:
  - Список опубликованных квестов (5 минут)
  - Информация о конкретном квесте (1 минута)
  - Статистика пользователя (30 секунд)
- [ ] Добавить инвалидацию кэша при изменениях
- [ ] Создать утилиты для работы с кэшем
